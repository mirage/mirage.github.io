core:

The core libraries that form Mirage. These are primarily module type definitions for functionality such as networking and storage, as well as the frontend configuration and CLI tooling.
  • mirage/mirage:

    This is the main repository that contains the Mirage type definitions and the CLI tool.

    opam: mirage, mirage-types, mirage-types-lwt, mirage-runtime
    docs: mirage, mirage-types, mirage-types-lwt, mirage-runtime
  • mirage/functoria:

    a DSL to describe a set of modules and functors, their types and how to apply them in order to produce a complete application.

    opam: functoria, functoria-runtime
    docs: functoria, functoria-runtime
  • mirage/mirage-block:

    generic operations over Mirage block devices.

    opam: mirage-block, mirage-block-lwt
    docs: mirage-block, mirage-block-lwt
  • mirage/mirage-protocols:

    a set of module types that libraries intended to be used as Mirage network implementations should implement.

    opam: mirage-protocols, mirage-protocols-lwt
    docs: mirage-protocols, mirage-protocols-lwt
  • mirage/mirage-entropy:

    Reliable entropy sources for Mirage unikernels.

    opam: mirage-entropy
    docs: mirage-entropy
  • mirage/mirage-time:

    Module types for time-related operations in Mirage.

    opam: mirage-time, mirage-time-lwt
    docs: mirage-time, mirage-time-lwt
  • mirage/mirage-random:

    Randomness signatures for Mirage, and an implementation using the OCaml stdlib.

    opam: mirage-random
    docs: mirage-random
  • mirage/mirage-net:

    Network (Ethernet) signatures for Mirage.

    opam: mirage-net, mirage-net-lwt
    docs: mirage-net, mirage-net-lwt
  • mirage/mirage-logs:

    a reporter for the Logs library that writes log messages to stderr, using a Mirage CLOCK to add timestamps.

    opam: mirage-logs
    docs: mirage-logs
  • mirage/mirage-kv:

    provides key/value store signatures that Mirage storage libraries can implement.

    opam: mirage-kv, mirage-kv-lwt
    docs: mirage-kv, mirage-kv-lwt
  • mirage/mirage-fs:

    provides filesystem module signatures that Mirage storage libraries can implement.

    opam: mirage-fs, mirage-fs-lwt
    docs: mirage-fs, mirage-fs-lwt
  • mirage/mirage-flow:

    Network flow implementations and combinators to manipulate and compose them.

    opam: mirage-flow, mirage-flow-lwt
    docs: mirage-flow, mirage-flow-lwt
  • mirage/mirage-console:

    Pure OCaml module types and implementations of Mirage consoles, for Unix and Xen.

    opam: mirage-console-lwt, mirage-console, mirage-console-unix, mirage-console-xen-backend, mirage-console-xen-cli, mirage-console-xen-proto, mirage-console-xen
    docs: mirage-console-lwt, mirage-console, mirage-console-unix, mirage-console-xen-backend, mirage-console-xen-cli, mirage-console-xen-proto, mirage-console-xen
  • mirage/mirage-clock:

    portable support for an operating system timesources.

    opam: mirage-clock, mirage-clock-freestanding, mirage-clock-lwt, mirage-clock-unix
    docs: mirage-clock, mirage-clock-freestanding, mirage-clock-lwt, mirage-clock-unix
  • mirage/mirage-channel:

    Channels are buffered reader/writers built on top of an unbuffered mirage-flow implementation.

    opam: mirage-channel, mirage-channel-lwt
    docs: mirage-channel, mirage-channel-lwt
  • mirage/mirage-stack:

    provides a set of module types which libraries intended to be used as Mirage network stacks should implement.

    opam: mirage-stack, mirage-stack-lwt
    docs: mirage-stack, mirage-stack-lwt
  • mirage/mirage-device:

    the signature for basic abstract devices for Mirage and a pretty-printing function for device errors

    opam: mirage-device
    docs: mirage-device
  • mirage/mirage-tc:

    a set of functors and combinators to convert to and from and JSON values and Cstruct buffers.

    opam: mirage-tc
    docs: mirage-tc
  • samoht/depyt:

    type combinators to define runtime representation for OCaml types and generic operations to manipulate values with a runtime type representation.

    opam: depyt
    docs: depyt

    opam: ocaml-freestanding
    docs: ocaml-freestanding
  • mirage/io-page:

    opam: io-page
    docs: io-page
  • mirage/parse-argv:

  • mirage/mirage-qubes:

Drivers for booting directly on the Xen hypervisor.

    Client and server implementations of the Xen paravirtualised block driver protocol

    opam: mirage-block-xen
    docs: mirage-block-xen
  • mirage/mirage-net-xen:

    Client and server implementations of the Xen paravirtualised network driver protocol

    opam: mirage-net-xen
    docs: mirage-net-xen
  • mirage/ocaml-vchan:

    implementation of the “vchan” shared-memory communication protocol.

    opam: vchan
    docs: vchan
  • mirage/mirage-bootvar-xen:

    library for reading Mirage unikernel boot parameters from Xen.

    opam: mirage-bootvar-xen
    docs: mirage-bootvar-xen

    a set of libraries for creating shared memory producer/consumer rings that follow the Xen hypervisor ABI for virtual devices.

    opam: shared-memory-ring
    docs: shared-memory-ring

    docs: evtchn

    opam: gnt
    docs: gnt
  • mirage/mirage-xen-minios:

    installs the C libraries for the Xen MiniOS and OpenLibM.

    opam: mirage-xen-minios
    docs: mirage-xen-minios
  • mirage/ocaml-xenstore:

    implementation of the Xenstore communication protocol, including client and server libraries.

    opam: xenstore
    docs: xenstore

    docs: shared-block-ring
  • mirage/ocaml-qcow:

    opam: qcow
